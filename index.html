// MZ1312 PROJECT DRIFTER.

Uncaged technology. Your own AI companion — local, scriptable, yours.

DRIFTER is a self-hosted, voice-first AI companion you actually control. No corporate kill switch, no rented intelligence, no data leaving the device unless you say so. Run it on your hardware, modify it like a Linux box, never depend on someone else's quarterly priorities.

01 / Features

Built to outlast the platform you're on.

Six load-bearing pieces. No subscriptions, no rate limits, no remote sunsets.

01
▶

Local-First Runtime

Runs entirely on your machine. CPU or GPU. Your audio, your conversations, your memory store — none of it touches a cloud unless you wire it up yourself.

02
◉

Voice-First Loop

Real conversation, not turn-based chat. Low-latency speech in, speech out. Interruptible. Designed to feel like talking to a person, not a search bar.

03
⌬

Persistent Memory

Your own memory directory. Flat files you can grep, version, and back up. Forget on demand. Export to anything. Never locked into someone else's vector database.

04
⌧

Scriptable Hooks

Drifter can call your shell, your code, your APIs. Schedule it, pipe to it, build skills as plain executables. It's a Unix citizen, not a walled garden.

05
◈

Pluggable Models

Local Llama, MLX, Whisper, Piper — or wire in a remote API when you want it. Swap the brain without rebuilding the body. Costs stay yours.

06
◐

No Kill Switch

You hold the binary, the weights, and the config. Nothing can be revoked from a dashboard 3,000 miles away. The technology stays uncaged.

02 / How It Works

Four steps, fully local.

01

Clone

Pull the repo. One binary, one config file, a few model weights. No installers, no accounts.

02

Configure

Pick your STT, TTS, and LLM. Local or remote. Edit one TOML file. Drifter respects your choices.

03

Run

drifter start — that's it. The voice loop is live and your memory store is being written.

04

Extend

Add a skill as a script. Hook into the bus. Drifter is a citizen of your shell, not a black box.
